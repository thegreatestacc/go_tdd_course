### Что нужно улучшить

| Приоритет | Проблема | Как улучшить |
|---|---|---|
| Высокий | `Repeat(letter, count)` использует константу `repeatCount` вместо аргумента `count` | Использовать `count` в цикле и добавить table-driven тесты для значений `0`, `1`, `3` и отрицательного числа |
| Высокий | `Hello` возвращает только имя при неизвестном языке, потому что prefix остаётся пустым | Использовать английский prefix по умолчанию и добавить отдельный тест |
| Высокий | Реализации `Add`, `Repeat`, `Sum`, `SumAll` и `SumAllTails` находятся в файлах `_test.go` | Перенести production-код в обычные `.go` файлы, оставив в `_test.go` только тесты и benchmarks |
| Средний | Имена пакетов `__integers` и `__iteration` не соответствуют обычному Go-стилю | Переименовать их в `integers` и `iteration` |
| Средний | Тесты проверяют мало вариантов входных данных | Использовать table-driven tests и добавить пустые слайсы, один слайс, несколько слайсов и нулевые значения |
| Средний | Проверки результатов дублируются, а слайсы сравниваются то через `slices.Equal`, то через `reflect.DeepEqual` | Вынести общие helpers и последовательно использовать `slices.Equal` для `[]int` |
| Низкий | В сообщениях тестов есть опечатки: `acutal`; тест `Add` дважды пишет `actual sum` | Исправить сообщения, чтобы падения тестов было проще диагностировать |
| Низкий | Некоторые имена недостаточно точны: `gettingPrefix`, `spanishHelloPerf`, общий `name` | Использовать `greetingPrefix`, `spanishHelloPrefix` и локальные данные тестов |

### Сильные стороны

- Все существующие тесты проходят на Go 1.26.4.
- Используются subtests, `t.Helper()`, benchmark и стандартный пакет `slices`.
- `SumAllTails` корректно обрабатывает пустой слайс.
- Код отформатирован через `gofmt` и остаётся небольшим и читаемым.

## Как считается прогресс

**Go fundamentals**, **Testing fundamentals** и **Build an application**:
всего 31 глава. Установочные, справочные и meta-главы в процент прогресса
не входят.
